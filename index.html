HTML

  • The very top of the html is the document type. It is not case sensitive.
  • The html opening tag represents the root, or start of the HTML document.
  • The head element contains information about the webpage. It is a container for metadata. It is not visible.
    • The meta element defines metadata such as the character set, description, keywords, author, and viewpoint.
    • The title element defines the title for a webpage as shown on the tab. It is required and is extremely important for Search Engine Optimization - SEO.
    • The style element defines the CSS styles associated with a website.
    • The base element defines the base URL for a webpage.
    • The link element connects an external resource to the HTML document.
    • The script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
    • The body element is known as the parent element. Elements nested inside other elements, such as the body element, are called child elements.
    • Headings go from h1 to h6 and represent the level of heading a given block of text represenets.
    • The main element contains the content of the body.
    • The section elements are semantic elements that define a section in a document by topics.
    • The p element represents a paragraph or block of text.
    • There are 2 types of lists:
      • ul is an unordered bulleted list
      • ol is an ordered numbered list.
        • Within the ul or ol is the li element.
        • the li elements are the list items or the bullet points/numbers.
    • The img element contains information about images that are displayed on a webpage. An attribute of the img element is the src attribute which defines the location of the image.
    • The br element creates a line break.
    • Hyperlinks are shown with the a element, or anchor, which creates links to the same webpage or other webpages. The href attribute points to the URL.
    • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

GIT

  • GIT commands
    • 1s: lists the files and directories on the computer
    • mkdir: creates a directory, or folder.
    • touch: creates a new file
    • start: opens the file
    • pwd: short for "print working directory" shows what's in the current directory
    • cd: change directories - cd bootcamp.
    • cd . .: navagate backwards one directory at a time.
    • git status: checks what branch we are currently on
    • git checkout -b branch-name: creates a new branch and switches to it
    • git add -A: the add command adds modifications in the current working branch to the staging area. the -A flag indicates that we want to add all changes.
    • git commit -m "add starter code": commits changes with a commit message describing what was changed.
    • git pull origin main: pulls down any changes from the base branch.
    • get push origin feature/starter-code: pushes changes to the remote branch.
    • git revert SHSnumber: reverts to a previous version.
  • Github flow:
    • Create a branch within the main branch.
    • Make additions or changes. Changes will not end up on the default branch until it's merged.
    • Make commits of all the changes with a commit message describing the additions or changes to the code. Commit often.
    • Push up the changes to GitHub repository.
    • Open a pull request. A pull request, or PR, is used to merge the branch into the base branch.
    • Other developers will then view the changes made. They may comment on it or request changes.
    • Once PR is approved, it is merged into the base branch.
