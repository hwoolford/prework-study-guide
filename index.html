HTML

  • The very top of the html is the document type. It is not case sensitive.
  • The html opening tag represents the root, or start of the HTML document.
  • The head element contains information about the webpage. It is a container for metadata. It is not visible.
    • The meta element defines metadata such as the character set, description, keywords, author, and viewpoint.
    • The title element defines the title for a webpage as shown on the tab. It is required and is extremely important for Search Engine Optimization - SEO.
    • The style element defines the CSS styles associated with a website.
    • The base element defines the base URL for a webpage.
    • The link element connects an external resource to the HTML document.
    • The script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
    • The body element is known as the parent element. Elements nested inside other elements, such as the body element, are called child elements.
    • Headings go from h1 to h6 and represent the level of heading a given block of text represenets.
    • The main element contains the content of the body.
    • The section elements are semantic elements that define a section in a document by topics.
    • The p element represents a paragraph or block of text.
    • There are 2 types of lists:
      • ul is an unordered bulleted list
      • ol is an ordered numbered list.
        • Within the ul or ol is the li element.
        • the li elements are the list items or the bullet points/numbers.
    • The img element contains information about images that are displayed on a webpage. An attribute of the img element is the src attribute which defines the location of the image.
    • The br element creates a line break.
    • Hyperlinks are shown with the a element, or anchor, which creates links to the same webpage or other webpages. The href attribute points to the URL.
    • The footer element appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation.

CSS

  • There are 3 ways to style a webpage using CSS:
    • Inline CSS
      • Used to style an element directly in the HTML file.
      • Inline styles apply to a specific HTML tag and styles a particular page element only.
      • Useful for quick and permanent changes but less flexible because each inline style must be edited individually whenever a design change is made.
    • Internal CSS style sheet
      • Contains CSS rules for the webpage in the head section of the HTML file.
      • The rules only apply to that page, and can be used to style multiple elements in the page code.A single change to the CSS rule applies to all tagged page elements.
    • External CSS style sheet
      • Most common way to style a webpage. It's preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
      • When using external style sheet, the file is linked using a link element placed in the HTML's head element.
  • Diagram of a CSS Rule:
    • Selector: defines the element or attributes to which the rules, or declaratioins, will apply.
    • Declarations: contain the CSS property to apply and the value of that property.
      • For example: in the declaration color:blue; the CSS property is color and the value is blue.
  • Using CSS
    • *selector: applies a rule to all elements visible on the page, such as margin and padding.
    • Link the CSS file to the HTML document by adding the code to the header before the title.
      • link element: creates the connection to the external file.
      • rel attribute: specifies the relationship between the current document and the linked document resource.
      • href attribute: specifies the URL of the external resource using a relative path.
    • Class attribute selector: Used to add styles to certain elements, but not all elements. Class attributes allow the sharing of a CSS rule to any element chosen by assigning the rule to a class attribute with a class selector.
    • .card: the period before card designates this as a class selector. Any HTML element assigned to the class, card, will gain these properties.
    • margin: if top/bottom are same and widths are the same, then first entry is for top and bottom and second entry is for left/right.
    • Margins and padding: A margin indicates how much space is around the outside of an element. A padding indicates how much space we want around the conent inside an element.

GIT

  • GIT commands
    • 1s: lists the files and directories on the computer
    • mkdir: creates a directory, or folder.
    • touch: creates a new file
    • start: opens the file
    • pwd: short for "print working directory" shows what's in the current directory
    • cd: change directories - cd bootcamp.
    • cd..: navagate backwards one directory at a time.
    • git status: checks what branch we are currently on
    • git checkout -b branch-name: creates a new branch and switches to it
    • git add -A: the add command adds modifications in the current working branch to the staging area. the -A flag indicates that we want to add all changes.
    • git commit -m "add starter code": commits changes with a commit message describing what was changed.
    • git pull origin main: pulls down any changes from the base branch.
    • get push origin feature/starter-code: pushes changes to the remote branch.
    • git revert SHSnumber: reverts to a previous version.
  • Github flow:
    • Create a branch within the main branch.
    • Make additions or changes. Changes will not end up on the default branch until it's merged.
    • Make commits of all the changes with a commit message describing the additions or changes to the code. Commit often.
    • Push up the changes to GitHub repository.
    • Open a pull request. A pull request, or PR, is used to merge the branch into the base branch.
    • Other developers will then view the changes made. They may comment on it or request changes.
    • Once PR is approved, it is merged into the base branch.

JavaScript

  • Basic JavaScript features
    • Variables: containers that store values. Start by declaring a variable with var container-name, followed by the equal sign, then the "name you give to the variable", followed by a semicolon which is used to separate statements on a single line.
    • Console log: the console.log method is used to output a message to the web console by adding an argument inside parenthesis.
    • Link .js files under the footer and above the closing element for body.
    • Use Chrome DevTools to view console.log output: right-click on the webpage and select Inspect. Select the console tab to view the output.
  • Control Flow
    • Control flow is the order in which a computer executes code in a script.
    • In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
    • Control flow can be interrupted using conditional statements and loops to the control the order in which code is executed, or if it is even executed at all.
  • Conditional logic
    • An if statement is a way of introducing decision-making into the code. When using the if statement, code will only execute IF the statement meets a condition, or is true.
    • Using else if allows for more conditions.
  • Arrays
    • An array is a single variable that is used to hold a group of data. Arrays arwe typically used to hold data that is related in some way.
    • To create an array, declare the variable using var, give the variable a name, and assign the variable a group of values inside brackets where each is inside quotes and there's a comma separating the values, ending with a semicolon. However, if different types of data is stored in an array, only use quotes around strings.
  • Using a for loop
    • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. Developers must use another statement to interrupt the control flow by executing a block of code a specific number of times.
    • Using a for loop tells the computer to go through each value in the array and log them to the console until there are no values left.
    • A for loop contains the following 3 important statements:
      • The first statement determines the starting point for the loop.
      • The second statement is the condition.
      • The third statement is what allows the array to iterate over each item.
